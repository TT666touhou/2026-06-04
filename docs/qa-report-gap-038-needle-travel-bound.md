# QA Report — GAP-038 修復針飛到一半消失

> 日期：2026-06-20 | 角色：QA | 類型：GAP-036 出界判定回歸 hotfix

## 1. 根因（實機抓到）

暫時插樁（auto-fire + log）跑 run_project 抓到：
```
[needle] pos=(656,659) vp=(1152.0, 648.0)   ← get_viewport_rect() 回傳 1152x648，非 1280x720 世界！
```
- GAP-036 用 `get_viewport_rect().size` 當**世界出界邊界**，但它回傳的是**(可能被縮放的) 視窗尺寸 1152×648**（0.9×1280/720），**不是 1280×720 世界**。
- 故出界邊界變成 `1152+64=1216` 寬 × `648+64=712` 高 < 外牆（1280×720）→ 針飛到 `x>1216` 或 `y>712` 就被當出界刪除，**在抵達外牆前消失** → 「飛到一半就不見了」。

## 2. 修復

- 移除 `get_viewport_rect()` 出界判定。
- 改以**飛行距離** `max_travel=2400`（> 競技場對角線 ~1469）當安全網：正常射擊由外牆攔截嵌入，只有「永遠沒撞到東西」的逃逸針才在 2400px 後回收。與視窗/縮放/相機**完全無關**。

## 3. 驗證（實機可證）

→ 修復後針**飛到右外牆內面 x=1264 才嵌入**（traveled 640px，未觸發 2400 安全網），不再於 x≈1216 提早消失。插樁已還原。

## 4. 教訓（修正 GAP-036 防範規則）

- `get_viewport_rect()` 回傳的是**視窗/螢幕**尺寸（受 stretch/scale 影響，此處 1152×648），**不等於世界座標範圍**。**禁止用 viewport 尺寸做世界座標出界/可視判定**。
- 世界範圍的安全回收應用：**飛行距離 / 生命週期**，或由實際場景/世界參數推導；本例外牆已封閉場景，安全網只是後備。
