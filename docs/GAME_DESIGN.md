## 🎮 1. 遊戲核心概念

### 1.1 核心定位 [DRAFT]
> 一款 **2D 橫向動作跑酷遊戲**，單人體驗。
> 玩家持有**固定數量的鋼針**，以投擲、穿線、鉤抓等方式同時解決移動與戰鬥。
> 資源管理（針的數量）是核心壓力，要求精確判斷每一根針的使用時機。

**設計靈感來源：**
- **Dimension W（漫畫）** — 主角 Kyouma 的戰鬥風格：非科技、純機械、近距投擲、精準打點
- **Titan Souls** — 一擊致命、武器（針）必須回收、Boss 弱點精確打擊

### 1.2 遊戲類型 [DRAFT]
- **主類型**：2D 橫向動作（Action Platformer）
- **次類型**：跑酷動作（Parkour Action）
- **人數**：單人
- **維度**：2D（橫向捲軸）

### 1.3 核心設計支柱 [DRAFT]
1. **資源即工具**：鋼針是移動工具也是武器，永遠有限，迫使玩家思考
2. **多功能單一機制**：同一根針根據使用方式產生完全不同的效果
3. **精確性挑戰**：移動與攻擊都需要瞄準與判斷，不靠連打取勝
4. **回收循環**：使用後必須回收針（或找到補充），製造節奏張力

### 2.3 細線（Wire）系統 [DRAFT]
**已確認：**
- 鋼索平台碰撞：單向平台（從下往上可穿透；下降時落上）；**S 鍵 = 穿透落下**（無需 Jump）[DRAFT]
- 鋼索「活化」視覺：兩針展開時線道外觀需有明顯變化（顏色/亮度）以提示可踩踏

**鋼索平台物理行為（2026-06-20 更新）[DRAFT]**
- **視覺下垂**：玩家站上平台後，catenary 曲線目標鬆弛量增加到 20px（`SAG_WEIGHT=20`），以 `lerpf` 速率 8.0 插值，使線條視覺向下凹陷；離開平台後以速率 3.0 回彈至零。
- **S 鍵穿透**：按下 S 鍵（`drop_through` action）後 0.25s 內關閉玩家對 layer 4 的碰撞感知，允許穿透落下；時間結束後自動恢復碰撞。
- **碰撞層分配**：`WirePlatform` 使用 collision layer 4（bitmask=8），不佔用 default layer 1；玩家 mask 加入 layer 4 才能站上平台。
- **方向正規化**：`_update_body()` 計算 wire 角度時，強制 `dir.x ≥ 0`，確保 `one_way_collision` 法向永遠朝世界上方（防止右→左錨點順序造成方向翻轉 GAP-027）。

**Wire 視覺規格（2026-06-20 更新）[DRAFT]**

線條顯示優先順序（高 → 低）：

| 優先 | 條件 | 視覺表現 | 顏色/線寬 |
|-----|------|---------|---------|
| P1 | 平台模式（anchor1 + anchor2 均存在） | 垂弧線：anchor1 ↔ anchor2；飛行中也持續顯示 | 亮金色 `(1.0, 0.92, 0.5)`、3px |
| P2 | 擺錘中 + 新針飛行中 | 直線：anchor1 → 飛行針尖端（延展視效） | 黃色 75% 透明、1.5px |
| P3 | 單針嵌入（擺錘模式，無飛行針） | **垂弧線（Catenary）**：Player → anchor1；鬆弛愈多弧度愈深（最大 60px 垂弧）；張力大時線條變白且加粗 | 黃色→白色（張力插值）、1.5→3px |
| P4 | 僅飛行中（無已嵌入錨點） | 直線：Player → 飛行針 | 黃色 60% 透明、1px |

**帶線針物理 — Verlet 繩 + 鐘擺擺盪（2026-06-20 重做，GAP-037）[DRAFT]**

> 問題：先前彈力繩的繩索視覺有「詭異的側向 S 彎曲」——因為渲染用 `slack = max_length - dist` 做**垂直於繩**的假鬆弛，繩近垂直時就往側邊鼓出，與物理（直線拉力）脫節。
> 重做方向：**視覺改用 Verlet 物理繩**（真實重力下垂、繃緊變直，無假彎曲）；**物理改回 GDD 的鐘擺長度約束**（盪繩），沿用 CharacterBody2D。
> 技術選型：研究後採 **Verlet integration**（業界 2D 繩首選，CRope2D 等亦同；支援 CharacterBody2D）。**不採 Joint2D 系列**——那需把 Player 改成 RigidBody2D，會棄掉現有平台移動/跳躍/慣性/鋼索平台等大量實作，風險過高（社群亦回報 Joint 抓鉤調校困難）。

- **Verlet 繩視覺**：繩由 N 個點組成，兩端釘在錨點與玩家，中間點以 Verlet 積分受重力下垂、距離約束維持節長。→ 鬆弛時自然垂落、繃緊時拉直，**不再有側向假彎曲**（視覺與物理分離，保持乾淨）。
- **自然鐘擺約束（2026-06-20 GAP-040 改回自然）**：繩有長度 `max_length`。
  - 距離 ≤ `max_length`：繩鬆，玩家自由落（純重力 + 空中控制）。
  - 距離 > `max_length`：繩繃緊 → **只約束距離**（把玩家夾回半徑圓上 + **消除朝外徑向速度**），玩家在重力下**自然擺盪**。
  - **不注入任何朝錨點的速度**——動量完全來自重力盪繩本身，**非人工 fling**（GAP-039 的彈簧加速度造成「特別添加的詭異動量」，已移除）。
- **自動收繩（位置約束，非彈簧）**：`max_length` 以 `auto_reel_speed` 快速縮短，把玩家拉向錨點；盪繩時收繩會**自然地**加快擺速（角動量守恆，如收手加快旋轉）——動量自然、不造假。
- **上甩接平台連段（自然版）**：用**盪繩累積的自然動量**（非人工 fling）→ 在擺盪高點射第二個帶線針 → 兩錨點成平台、繩釋放 → 帶**自然**動量站上平台。
- **E（加速收繩）**：按住 E = 額外加速縮短繩長。**Q 切斷**（同 §2.4）。
- **保留動量 + 空中控制**：在繩上時左右輸入為加速度疊加（可 pump 擺盪），不覆寫速度。
- **可調參數（@export）**：`auto_reel_speed`(收繩快慢)、`min_rope_length`、`reel_speed`(E)、`swing_accel`、`swing_air_drag`。（已移除彈簧 `rope_stiffness/damping/max_accel`。）

**第三針在平台模式下的行為** [DRAFT]
- 射出第三根 wire needle → 落點（anchor3）成為玩家新的擺錘錨點
- 舊平台（anchor1↔anchor2）繼續存在，碰撞與視覺均保留
- 玩家繩子從 anchor3 出發，可繼續 E 收線或盪繩
- **平台不移位，也不解散**；anchor3 只是取代玩家的繩子起點

### 2.4 Q 斷線 與 F 回收規則（2026-06-20 更新，GAP-032）[DRAFT]

**Q（斷線）— 只切「當前與玩家相連的擺錘線」**
- Q 只能切斷**當前正連接玩家的擺錘線**（active pendulum wire）。
- 切斷後：玩家與該針脫離，**針仍留在地形**（成為待回收的無連接 wire 針），可事後用 F 回收。
- **已成平台的線無法用 Q 切斷**：當兩針已展開為平台（無 active 擺錘線）時，按 Q **無作用**。
- 要拆掉平台，必須**先用 F 回收平台的其中一端**（見下方優先級），平台解散後另一端自動轉回擺錘線。

**F（回收）— 依優先級擇一回收**
- F 一次只回收**一根**針，依下列優先級從高到低擇一：

| 優先 | 對象 | 說明 | 距離限制 |
|-----|------|------|---------|
| 1（最高） | **無牽線的鋼針** | 攻擊針（沒有細線） | 需在回收半徑內 |
| 2 | **與玩家相連的鋼針** | 當前 active 擺錘線的錨點 | 需在回收半徑內 |
| 3（最低） | **建立平台的鋼針** | 平台的兩個端點之一 | 需在回收半徑內 |

- **所有針一律要夠近才能撿**（2026-06-20 GAP-034 修正）：擺錘針不再「不限距離」，與其他針相同，需在回收半徑內。要回收遠處的擺錘針，需先盪近、或 Q 斷線後走過去。
- **回收半徑**：`retrieve_radius` 由 30px 提高到 **60px**（2026-06-20 GAP-034）。原因：距離以玩家**中心**量測，而玩家高 64px、中心在腳上方約 32px，30px 半徑永遠碰不到落在腳邊/平台面的針 → 平台針撿不起來。60px 才足以涵蓋腳邊的針。
- 同一優先級內有多個候選時，取**最近**者。
- 回收平台端點（優先級 3）→ 平台解散；若仍有剩餘 wire 針，自動轉為玩家擺錘線（沿用 GAP-029 行為）。
- 設計意圖：攻擊針最先清、擺錘針次之、平台針最後才動（避免攻擊針在旁時誤拆平台）。

### 2.5 回收提示 UI（2026-06-20 新增，GAP-033）[DRAFT]

> 痛點：玩家難以判斷何時靠得夠近、哪根針會被 F 收回（尤其平台針）。

- **世界座標文字提示**：在「當前可回收」的鋼針上方顯示文字標籤（如 `[F] 平台針`）。
- **目標高亮**：實際會被 F 收回的那根（最高優先級）以**亮色**顯示，其餘可回收針以**暗色**顯示，讓玩家一眼確認 F 的目標。
- **顯示條件**：僅對「可回收」的針顯示（**所有針一律需在回收半徑 60px 內**，GAP-034 後無例外）。
- **UI 架構（可復用）**：拆為兩層 — 通用世界文字標籤元件 + 提示控制器；後期可加 tween（淡入/彈跳）、圖示、顏色主題而不動呼叫端。
- **資源位置**：`scripts/ui/`、`scenes/ui/`。

**技術規格**
- 線條實作：8 段 Line2D（Player.WireRenderer），`top_level=true`（世界座標）
- 垂弧計算：`sag = min(slack × 0.35, 60.0)`，各中間點加 `sag × sin(π × t)` 偏移（重力垂直分量正確）
- WirePlatform 節點本身不含視覺；所有線條由 Player 的 WireRenderer 統一管理

> 線的物理特性尚待決定：
- 線的類型：剛性 / 彈性 / 切割型 / 可傳導（電/熱）？
- 線可掛幾根針？是否有張力上限？
- 線是否會自然消耗或斷裂？

## 9. MVP 原型規格（最小可遊玩版本）

> MVP 目標：用純幾何形狀（ColorRect / CircleShape2D）驗證整體遊玩手感。
> 所有數值為初始設計值，進入 Godot 後可快速調整。

### 9.1 平台與操控 [DRAFT]

| 項目 | 規格 |
|------|------|
| **目標平台** | PC（鍵盤 + 滑鼠） |
| **移動** | A/D 左右移動（**含加速/減速慣性**，詳見 §9.5） |
| **跳躍** | Space（**含 jump buffer + coyote time + 可變跳躍高度**，詳見 §9.5） |
| **預留擴充** | W 鍵留給未來技能（如向上鉤抓） |
| **瞄準方式** | 滑鼠游標方向（玩家角色指向游標） |
| **投擲觸發（攻擊針）** | 左鍵點擊 = 投出無線針（攻擊用） |
| **投擲觸發（帶線針）** | **單按右鍵** = 投出帶細線的針（GAP-039 簡化）|
| **帶線針行為** | 命中即建立錨點並**自動把玩家彈性拉過去**（彈力繩，詳見 §2.3） |
| **加速拉近（選用）** | E 鍵按住 = 額外朝錨點加速，更快拉到錨點旁 |
| **斷線** | Q 鍵 = 切斷當前帶線針（針留在地形，可事後回收）；已成平台的線無法用 Q 切斷（詳見 §2.4） |
| **回收針** | F 鍵 = 依優先級回收（無線針 > 與玩家相連針 > 平台針；詳見 §2.4） |
| **帶線針物理** | 彈力繩：恆定朝錨點彈性拉力 + 低阻尼回彈（詳見 §2.3）|
| **飛行中調整** | 不允許（投出後方向固定） |

### 9.5 移動手感與 2D 必備機制（2026-06-20 新增，GAP-035）[DRAFT]

> 目標：地面/空中操控更紮實，補上現代 2D 平台遊戲的標準「手感」機制。

- **左右移動慣性**：水平移動改為**加速/減速**（非瞬間到速度），地面與空中分別有不同加速度與摩擦：
  - `ground_accel` / `ground_friction`（地面起步較快、放開即減速）
  - `air_accel` / `air_friction`（空中操控較弱、保留動量）
- **Jump Buffer（跳躍緩衝）**：落地前 `jump_buffer_time`（約 0.1s）內按跳躍，落地瞬間自動執行 → 不吃輸入。
- **Coyote Time（土狼時間）**：離開地面 `coyote_time`（約 0.1s）內仍可起跳 → 邊緣起跳更寬容。
- **可變跳躍高度**：跳躍上升中放開跳鍵 → 立即削減上升速度（短按小跳、長按高跳）。
- **Player 性能微調**：`move_speed`、`jump_velocity` 略為提升（手感更俐落）。
- **可調參數（@export）**：上述各項皆開放，供手感微調。
