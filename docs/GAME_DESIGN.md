## 🎮 1. 遊戲核心概念

### 1.1 核心定位 [DRAFT]
> 一款 **2D 橫向動作跑酷遊戲**，單人體驗。
> 玩家持有**固定數量的鋼針**，以投擲、穿線、鉤抓等方式同時解決移動與戰鬥。
> 資源管理（針的數量）是核心壓力，要求精確判斷每一根針的使用時機。

**設計靈感來源：**
- **Dimension W（漫畫）** — 主角 Kyouma 的戰鬥風格：非科技、純機械、近距投擲、精準打點
- **Titan Souls** — 一擊致命、武器（針）必須回收、Boss 弱點精確打擊

### 1.2 遊戲類型 [DRAFT]
- **主類型**：2D 橫向動作（Action Platformer）
- **次類型**：跑酷動作（Parkour Action）
- **人數**：單人
- **維度**：2D（橫向捲軸）

### 1.3 核心設計支柱 [DRAFT]
1. **資源即工具**：鋼針是移動工具也是武器，永遠有限，迫使玩家思考
2. **多功能單一機制**：同一根針根據使用方式產生完全不同的效果
3. **精確性挑戰**：移動與攻擊都需要瞄準與判斷，不靠連打取勝
4. **回收循環**：使用後必須回收針（或找到補充），製造節奏張力

### 1.4 遊戲名稱 [DRAFT]
> **TBD** — 目前工作目錄名 "ANTIGRACITY" 僅為 IDE 識別名，非正式遊戲名稱。
> 命名於下一輪設計討論中確定。

### 2.3 細線（Wire）系統 [DRAFT]
**已確認（2026-06-20 GAP-041 大幅簡化）：**
- **只保留盪繩抓鉤**；**平台機制、S 鍵穿透、Q/E/F 全部移除**。

**帶線針物理 — 右鍵按住抓鉤 + 自動收繩 + 自然鐘擺（GAP-041）[DRAFT]**

- **右鍵按住發射**：按住右鍵射出帶線針抓鉤；命中即建立錨點。
- **自動收繩**：命中後 `max_length` 以 `auto_reel_speed` 自動縮短（位置約束把玩家拉向錨點；盪繩時角動量守恆自然加速）。
- **自然鐘擺約束**：距離 ≤ `max_length` 鬆弛自由落；> `max_length` 繃緊 → 只夾位置到半徑圓 + **消除朝外徑向速度** → 自然擺盪。**不注入任何速度**（動量純來自重力盪繩，GAP-040）。
- **鬆開右鍵 → 自動斷開 + 回收**：放開右鍵**立即斷繩**，且該帶線針**一併回收**（返回持有數量）；若在飛行途中放開亦取消並回收。
- **Verlet 繩視覺**：繩兩端釘在錨點與玩家，重力下垂、繃緊變直，無側向假彎曲（GAP-037）。飛行中：直線 Player → 飛行針。
- **可調參數（@export）**：`auto_reel_speed`、`min_rope_length`、`swing_accel`、`swing_air_drag`、`rope_segments`。

### 2.4 鋼針回收規則（2026-06-20 GAP-041 大幅簡化）[DRAFT]

> **移除 Q / E / F 三個按鍵功能**，改為情境自動化：

- **帶線針（盪繩）**：**鬆開右鍵 → 自動斷繩，且該針一併回收**（取代舊 Q 斷線 + F 回收，詳見 §2.3）。
- **攻擊針（左鍵）**：射出後留在地形；**玩家靠近（在回收半徑 `retrieve_radius=60px` 內）即自動回收**，不需按鍵（取代舊 F）。
- **回收半徑 60px**：以玩家**中心**量測；玩家高 64px、中心在腳上方約 32px，故 60px 才涵蓋腳邊的針（GAP-034 沿用）。
- 已移除：Q（斷線）、E（加速收繩）、F（手動回收）、優先級系統、平台回收。

### 2.5 回收提示 UI — 已移除（2026-06-20 GAP-041）

> 該 UI（GAP-033）是為「F 手動回收」顯示可回收提示。GAP-041 移除 F、攻擊針改自動回收後，**提示 UI 已停用**（player 不再實例化）。`scripts/ui/`、`scenes/ui/` 的 `WorldLabel`/`PickupPromptUI` 元件保留為**可復用資產**，未來需要再接回。

**技術規格**
- 線條實作：8 段 Line2D（Player.WireRenderer），`top_level=true`（世界座標）
- 垂弧計算：`sag = min(slack × 0.35, 60.0)`，各中間點加 `sag × sin(π × t)` 偏移（重力垂直分量正確）
- WirePlatform 節點本身不含視覺；所有線條由 Player 的 WireRenderer 統一管理

> 線的物理特性尚待決定：
- 線的類型：剛性 / 彈性 / 切割型 / 可傳導（電/熱）？
- 線可掛幾根針？是否有張力上限？
- 線是否會自然消耗或斷裂？

### 9.5 移動手感與 2D 必備機制（2026-06-20 新增，GAP-035）[DRAFT]

> 目標：地面/空中操控更紮實，補上現代 2D 平台遊戲的標準「手感」機制。

- **左右移動慣性**：水平移動改為**加速/減速**（非瞬間到速度），地面與空中分別有不同加速度與摩擦：
  - `ground_accel` / `ground_friction`（地面起步較快、放開即減速）
  - `air_accel` / `air_friction`（空中操控較弱、保留動量）
- **Jump Buffer（跳躍緩衝）**：落地前 `jump_buffer_time`（約 0.1s）內按跳躍，落地瞬間自動執行 → 不吃輸入。
- **Coyote Time（土狼時間）**：離開地面 `coyote_time`（約 0.1s）內仍可起跳 → 邊緣起跳更寬容。
- **可變跳躍高度**：跳躍上升中放開跳鍵 → 立即削減上升速度（短按小跳、長按高跳）。
- **Player 性能微調**：`move_speed`、`jump_velocity` 略為提升（手感更俐落）。
- **可調參數（@export）**：上述各項皆開放，供手感微調。
